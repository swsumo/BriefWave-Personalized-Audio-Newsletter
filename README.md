# 🎙️ BriefWave — Hyper-Personalized AI Audio Newsletter

> **Your daily India news briefing. Generated by AI. Delivered as a podcast.**

BriefWave is a fully automated, end-to-end AI pipeline that generates a personalized, radio-style audio news briefing based on your selected interests.

Choose your categories.
Choose your depth.
Get a professionally narrated MP3.

Built entirely using **n8n orchestration**, **Google Gemini 2.5 Flash**, **GNews API**, **ElevenLabs TTS**, and **Supabase** — with zero traditional backend code.

---

## 🚀 What Makes BriefWave Special?

Unlike typical news apps, BriefWave:

* Fetches **real headlines from yesterday**
* Filters by **user-selected categories + sub-interests**
* Generates a **structured radio broadcast script**
* Converts it into **natural AI voice audio**
* Stores both transcript and MP3 in the cloud
* Plays it back instantly in the browser

All via a no-code workflow.

## 🏗️ System Architecture

```
Frontend (index.html)
        │
        │  POST { categories, subInterests, length, date }
        ▼
   n8n Webhook
        │
        ▼
   HTTP Request → GNews API (India headlines)
        │
        ▼
   Code Node → Category filtering & formatting
        │
        ▼
   Gemini 2.5 Flash → Structured radio script
        │
        ▼
   ElevenLabs → Text-to-Speech (MP3)
        │
        ▼
   Supabase
      ├── Store transcript (Postgres)
      └── Store audio file (Storage)
        │
        ▼
   Respond to Webhook
        │
        ▼
Frontend Audio Player + Transcript
```

---

## 2️⃣ Supabase Setup

### Create Database Table

Run inside Supabase SQL Editor:

```sql
create table briefings (
  id uuid primary key default gen_random_uuid(),
  created_at timestamptz default now(),
  date date,
  categories text[],
  length_mode text,
  transcript text,
  audio_url text
);
```

### Create Storage Bucket

1. Go to Storage → New Bucket
2. Name: `audio-briefings.`
3. Set bucket to **Public**

---

## 3️⃣ n8n Workflow

Node Chain:
```
Webhook → HTTP Request → Code → AI Agent → ElevenLabs → Supabase → Respond to Webhook
```
### 🔹 webhook

* Method: POST
* CORS: *
* Respond using Respond to Webhook node
### 🔹 HTTP Request (GNews)
* GET → [https://gnews.io/api/v4/top-headlines](https://gnews.io/api/v4/top-headlines)
* Params:
  * token
  * lang=en
  * country=in
  * max=20
  * from / to (dynamic date logic)
### 🔹 Code Node
* Filters by category
* Applies keyword mapping
* Formats structured text for Gemini
### 🔹 AI Agent
* Model: Gemini 2.5 Flash
* Memory: Postgres Chat Memory
* Strict word count + format rules
### 🔹 ElevenLabs
* Text → Speech
* Voice: Sarah (or any)
* Output: MP3 Binary
### 🔹 Supabase
* Save transcript to DB
* Upload MP3 to Storage
